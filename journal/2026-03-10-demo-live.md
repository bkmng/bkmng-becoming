## What was observed

`result-working-demo` is now realised. The Becoming demo is live at
`app.bkmng.com` with three navigable views: Decisions, Assumptions,
and a relation graph (Cytoscape.js). Data is loaded at build time from
this repository as a git submodule.

IAP is active — access requires a Google account explicitly granted
`roles/iap.httpsResourceAccessor`. Currently only `frode@bkmng.no`.

CI/CD is working: pushing to `bkmng/bkmng-app:main` triggers a
Cloud Build pipeline (~2.5 min) that builds a Docker image, pushes to
Artifact Registry, and deploys to Cloud Run.

## What this confirms

- A static Astro app is sufficient for the demo at this stage — no platform
  API or database needed (assumption-d005-3: confirmed)
- Infrastructure-as-code scripts made GCP setup manageable (assumption-d005-1: confirmed)
- The demo with real content is compelling enough to proceed to NHH engagement
  (assumption-d002-1: early confirmation — needs external validation)
