# Decision 001 — Use Becoming to document the BKMNG project itself

**Date:** 2026-03-10  
**Status:** Active  
**Made by:** Founding team

---

## Context

BKMNG is building a tool for explicit organizational self-representation. The question arose: should we use the tool for our own strategic work, or wait until it is mature enough?

## Options considered

1. Wait until the platform is built before using Becoming for internal work
2. Use Becoming immediately, even in document form, to drive both product thinking and organizational practice

## Decision

Use Becoming immediately to document the BKMNG project itself — starting with this repository.

`bkmng-becoming/` is the first instance of a Becoming model: BKMNG's own evolving strategy, represented using Becoming primitives.

## Rationale

1. This is the most direct form of "drinking your own champagne"
2. It produces authentic demo content — the demo *is* this project
3. It creates feedback on the model's usefulness before the platform is built
4. It keeps the project honest: if Becoming cannot represent its own strategic work, the model needs to improve

## Assumptions

- **Actor assumption:** The founding team will find Becoming-structured documentation more useful than unstructured prose over time
- **Activity assumption:** Writing decisions and journal entries in Becoming format will surface model gaps faster than theorizing in isolation
- **Result assumption:** This repo will serve as compelling evidence of the model's value for NHH and other collaborators

## Expected results

- A growing body of Becoming-formatted strategic records
- Identified gaps and open questions that improve the formal model
- A demo narrative that shows Becoming working on real strategy

## Commits to

- `bkmng-becoming/` as a live Becoming instance
- All major project decisions recorded here going forward
- Regular journal entries covering observations and shifts in understanding
